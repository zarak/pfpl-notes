1 Abstract Syntax Trees

  • An abstract syntax tree, or ast for short, is an ordered tree whose leaves are variables and whose interior nodes are operators whose arguments are its children.
  • Abstract syntax trees are classified into a variety of sorts corresponding to different forms of syntax.
  • A variable stands for an unspecified, or generic, piece of syntax of a specified sort.
    • Here variables range over ast’s of a specified sort.
    • A variable is given meaning by substition.
  • Ast’s may be combined by an operator, which has both a sort and an arity, a finite sequence of sorts specifying the number and sorts of its arguments.
  • The tree structure of ast’s supports a very useful principle of reasoning, called structural induction.
  • sort Exp example to prove some property \(\mathcal P(a)\) holds for all ast’s \(a\) of sort Exp:
    1. the property holds for any variable \(x\) of sort Exp.
    2. the property holds for any number num[n]: For every \(n \in \mathbb{N}; \mathcal{P}(\text {num}[n])\)
    3. assuming the property holds for \(a_1\) and \(a_2\), it holds for \(\text{plus} (a_1; a_2)\) and \(\text{times} (a_1; a_2)\): If \(\mathcal P(a_1)\) and \(\mathcal P(a_2)\), then \(\mathcal P (\text{plus}(a_1; a_2))\) and \(\mathcal P (\text {times} (a_1; a_2))\).
  • Precise definitions
    • Let \(S\) be a finite set of sorts.
    • Let \(\{ \mathcal O_s \}_{s \in S}\) be a sort-indexed family of operators o of sort \(s\) with arity \(\text{ar} (o) = (s_1, \cdots, s_n)\).
    • Let \(\{ \mathcal X_s \}_{s \in S}\) be a sort-indexed family of variables x of sort \(s\).
    • The family \(\mathcal A[\mathcal X] = \{ \mathcal A [\mathcal X]_s \}_{s \in S}\) of ast’s of sort \(s\) is defined as follows:
      1. A variable of sort \(s\) is an ast of sort \(s\): If \(x \in \mathcal X_s\), then \(x \in \mathcal A[\mathcal X]_s\).
      2. Operators combine ast’s: If \(o\) is an operator of sort \(s\) such that \(\text{ar}(o) = (s_1, \cdots, s_n)\) and if \(a_1 \in \mathcal A[\mathcal X]_{s_1}, \cdots, \mathcal A[\mathcal X]_{s_n}\), then \(o(a_1; \cdots ; a_n) \in \mathcal A [\mathcal X]_s\).
  • To show \(\mathcal P(a)\) holds for every \(a \in \mathcal A[\mathcal X]\), it is enough to show
    1. If \(x \in \mathcal X_s\), then \(\mathcal P_s(x)\).
    2. If \(o\) has arity \((s_1, \cdots, s_n)s\) and \(\mathcal P_{s_1}(a_1), \cdots, \mathcal P_{s_n}(a_n)\) then \(\mathcal P_s(o(a_1; \cdots ; a_n)\).
  • Substitution is defined by the following equations:
    1. \([b/x]x = b\) and \([b/x]y = y\) if \(x \neq y\).
    2. \([b/x]o(a_1 ; \cdots ; a_n) = o([b / x]a_1 ; \cdots ; [b / x] a_n)\).

1.1 Theorem

If \(a \in \mathcal A[ \mathcal X, x]\), then for every \(b \in \mathcal A[\mathcal X]\) there exists a unique \(c \in \mathcal A[\mathcal X]\) such that \([b/x]a=c\).

1.1.1 Proof

By structural induction on \(a\). If \(a = x\), then \(c = b\) by definition; otherwise, if \(a = y \neq x\), then \(c = y\) also by definition. Otherwise, \(a = o(a_1, \cdots, a_n)\), and we have by induction unique \(c_1, \cdots, c_n\) such that \([b/x]a_1 = c_1\) and \(\cdots [b/x]a_n = c_n\), and so \(c\) is \(c = o(c_1 ; \cdots ; c_n)\) by definition of substitution.

2 Abstract Binding Trees

  • Abtract binding trees or abt’s enrich ast’s with the means to introduce new variables and symbols, called a binding, with a specified range of significance, called its scope.
  • Example \[\text{let } x \text{ be } a_1 \text{ in } a_2\] introduces a variable \(x\) for use within the expression \(a_2\) to stand for the expression \(a_1\).
  • An argument to an operator is called an abstractor and has the form \(x_1, \cdots, x_k.a\). The sequence of variables \(x_1, \cdots, x_k\) are bound within the abt \(a\).
  • To account for binding, operators are assigned generalized arities of the form \((\upsilon_1, \cdots, \upsilon_n)s\), which specifies operators of sort \(s\) with \(n\) arguments of valence \(\upsilon_1, \cdots, \upsilon_n\).
    • In general, a valence \(\upsilon\) has the form \(s_1, \cdots, s_k.s\) which specifies the sort of the argument as well as the number of and sorts of the variables bound within it.

3 Exercises

3.1 Exercise 1.1

Prove by structural induction on abstract syntax trees that if \(\mathcal X \subseteq \mathcal Y\), then \(\mathcal A[\mathcal X] \subseteq \mathcal A[\mathcal Y]\).

3.1.1 Proof

Let \(a \in \mathcal A[ \mathcal X]\). \(a\) is either a variable or an operator. Suppose \(a\) is a variable. Then if \(a \in \mathcal X_s\), for some sort \(s \in \mathcal S\), then \(a \in Y_s\) by hypothesis, and hence \(a \in \mathcal A[\mathcal Y]_s\).

Next, suppose \(a\) is an operator \(o\), and let \(\text{ar}(o) = (s_1, \cdots, s_n)s\). If \(a_1 \in \mathcal A[\mathcal X]_{s_1}, \cdots, \mathcal A[\mathcal X]_{s_n}\), then by induction \(a_1 \in \mathcal A[\mathcal Y]_{s_1}, \cdots, \mathcal A[\mathcal Y]_{s_n}\), and hence by definition \(o(a_1; \cdots ; a_n) \in \mathcal A [\mathcal Y]\).
